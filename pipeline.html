Pipeline en scikit-learn

Un Pipeline en scikit-learn es una herramienta que permite encadenar mltiples etapas de procesamiento y modelado de datos dentro de un nico flujo. Esto incluye desde la preparacin de datos (limpieza, transformacin, seleccin de caractersticas) hasta el entrenamiento y prediccin con un modelo final. Su uso principal es garantizar que todos los pasos del flujo se apliquen de forma consistente tanto en los datos de entrenamiento como en los de prueba, evitando problemas de fuga de datos y asegurando reproducibilidad. Adems, al encapsular todo en un solo objeto, el Pipeline puede integrarse fcilmente con tcnicas de ajuste de hiperparmetros como GridSearchCV o RandomizedSearchCV, permitiendo optimizar parmetros de cualquier etapa usando una sintaxis uniforme.

En un Pipeline tpico, se pueden incluir transformadores como FunctionTransformer para aplicar lgica personalizada (por ejemplo, extraccin de ao, mes y da desde una fecha), imputadores como SimpleImputer para manejar valores faltantes, escaladores como StandardScaler para normalizar variables numricas y codificadores como OneHotEncoder para variables categricas. Para manejar diferentes tipos de variables simultneamente, se suele utilizar ColumnTransformer, que permite definir sub-pipelines especficos para columnas numricas y categricas. Tambin es comn integrar pasos de filtrado o seleccin de variables, como SelectKBest. Finalmente, el estimador final (modelo de Machine Learning) puede ser cualquier algoritmo compatible con scikit-learn, por ejemplo, RandomForestRegressor o LogisticRegression.

def pre_base_ops(X: pd.DataFrame) -> pd.DataFrame:
    X = X.copy()

    if "Release Date" in X.columns:
        fecha = pd.to_datetime(X["Release Date"], errors="coerce")
        X["release_year"]  = fecha.dt.year
        X["release_month"] = fecha.dt.month
        X["release_day"]   = fecha.dt.day
        X = X.drop(columns=["Release Date"])

    for col in X.columns:
        if X[col].dtype == "object":
            s = X[col].astype(str).str.replace(",", "", regex=False).str.replace(" ", "", regex=False)
            conv = pd.to_numeric(s, errors="coerce")
            ratio = 1.0 - conv.isna().mean()
            if ratio >= 0.7:  
                X[col] = conv

    return X

pre_base = FunctionTransformer(pre_base_ops, validate=False)

cat_encoder = OneHotEncoder(handle_unknown="ignore", min_frequency=10)

num_pipe = Pipeline(steps=[
    ("imputer", SimpleImputer(strategy="median")),
    ("scaler", StandardScaler())
])

cat_pipe = Pipeline(steps=[
    ("imputer", SimpleImputer(strategy="most_frequent")),
    ("onehot", cat_encoder),
])

preprocessor = ColumnTransformer(
    transformers=[
        ("num", num_pipe, selector(dtype_include=np.number)),
        ("cat", cat_pipe, selector(dtype_include=object)),
    ],
    remainder="drop",
    verbose_feature_names_out=False
)

Relacin con las primeras etapas de CRISP-DM

En este proyecto, el uso de un Pipeline se enmarca dentro de las dos primeras fases del modelo CRISP-DM: Business Understanding y Data Understanding. En la fase de Business Understanding se defini como objetivo predecir la popularidad de canciones en Spotify para priorizar estrategias de promocin y optimizar recursos de marketing. Se estableci que el criterio de xito sera alcanzar una mtrica R aceptable (en el caso de regresin) o una precisin adecuada (en el caso de clasificacin), teniendo en cuenta restricciones como disponibilidad de datos y tiempo de cmputo. Asimismo, se identificaron riesgos como la posible presencia de datos desbalanceados, sesgos derivados de la popularidad histrica y valores extremos en las mtricas de streaming.

En la fase de Data Understanding, se analiz la estructura del dataset, identificando tipos de variables (numricas, categricas, fechas) y explorando su distribucin y calidad. Se detectaron problemas como duplicados en las columnas clave (Track, Artist, Release Date), valores numricos almacenados como texto y formatos inconsistentes en fechas. A partir de este anlisis, se tomaron decisiones de preparacin de datos: eliminacin de duplicados y filas sin valor objetivo, creacin de variables derivadas como release_year, release_month y release_day, conversin de variables numricas mal tipadas y uso de un ColumnTransformer para aplicar pipelines especficos a cada tipo de dato. Estas decisiones fueron fundamentales para garantizar que el Pipeline tuviera entradas limpias y consistentes, y que el modelo final recibiera informacin adecuada para el entrenamiento.
